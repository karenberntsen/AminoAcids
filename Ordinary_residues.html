Ordinary residues

This data set consists of residues in structures solved by X-ray diffraction methods and which have a score of 1.0 points according to the WHAT IF command WSVUDB.
The criteria use are explained in the WSDL and the HELP file of the webservices that are located at http://wiws.cmbi.ru.nl/wsdl/ and http://wiws.cmbi.ru.nl/help/, respectively. The web-service UseresidueDB gives scores to the atoms in a residue. The score for the residue is then the sum of all atomic scores.

UseResidueDB

Use residue in database or not

WHAT IF command: WSVUDB

Description: Returns for each residue a value that tells you whether you can use this residue in database applications or that you better do not use it. The values are up to 1.0. 1.0 means that this is a perfect residue to use in training sets, in statistics, in databases, etcetera.
Each residue starts with 1.0 points. After that points are subtracted:

  • -0.75 Bad Cα
  • -0.50 Bad other backbone atom
  • -0.60 Bad Cβ (not for Gly)
  • -0.40 One or more other bad side chain atoms
  • -0.35 N- or C-terminal residue (or next to invisible residue)
  • -0.25 Next to residue from previous line
  • -0.40 For non-canonical residue
  • -0.40 For Gly or Pro
  • -0.10 Next to one of previous two lines
  • -0.15 Cα has high B-factor
  • -0.10 Other backbone atoms has high B-factor
  • -0.08 Cβ has high B-factor

A B-factor is called high if it is either >60.0, or > 2.5 times the average B-factor of all Cα's in the whole file. An atom is called bad if it is absent, if any bond-length to a covalently bound atom deviates more than 0.4 Angstrom from the ideal value, and (for this service only) if anything non-canonical is bound to it. This latter criteria is more relevant for cysteines and serines than for isoleucines, of course, but it also catches situations where clashes with other groups are so severe that the software assumes a covalent bond.
