Experiments are YASARA's way of handling complicated things at the touch of a button. During an experiment, YASARA takes over control. As in real life, you should not interfere too much while YASARA is cooking. E.g. if you try to delete some atoms or change the simulation parameters in the middle of a 'Neutralization Experiment', YASARA will ask you to stop the experiment first. Like molecular dynamics simulations, experiments cannot be undone.

When you start an experiment ('Experiment On'), YASARA performs various checks and reassigns the force field parameters. If you want to modify the force field, e.g. by adding distance constraints or modifying atom charges, do that immediately after starting the experiment, not before. Otherwise the changes may be undone.

The 'Experiment Pause' command only pauses YASARA's control over the scene, e.g. if a simulation is running as part of the experiment, it will continue running until it is paused separately or until YASARA is paused completely by pressing the <Pause> key.

Example 1:
Experiment Minimization

Prepare an energy minimization experiment, see example macro below. YASARA creates a cell and starts with a steepest descent minimization using the current simulation parameters. Regions with strong bumps are initially minimized without electrostatic interactions to avoid short-range energy traps. As soon as the bumps have been removed and the worst conformational stress is gone (i.e. when the square root of MinStep/AccelMax becomes greater than 2.5 fs, with AccelMax being the maximum acceleration acting on any atom in the soup), a simulated annealing minimization is done until the energy (calculated every 200 steps) converges. The convergence criterion can be set with the 'Convergence' parameter. This procedure thus moves the soup to a deep energy minimum in the neighborhood, which is not necessarily the global energy minimum. To describe this method in a materials & methods section, the following text could be used: "To remove bumps and correct the covalent geometry, the structure was energy-minimized with the .... force field [ref ], using a ... A force cutoff and (if applied) the Particle Mesh Ewald algorithm [ref] to treat longrange electrostatic interactions. After removal of conformational stress by a short steepest descent minimization, the procedure continued by simulated annealing (time step 2 fs, atom velocities scaled down by 0.9 every 10th step) until convergence was reached, i.e. the energy improved by less than 0.05 kJ/mol per atom during 200 steps.".

Example 2:
Experiment Neutralization

Prepare a cell neutralization and pKa prediction experiment, see example macro below. It is helpful to optimize the hydrogen bonding network before. YASARA then predicts pKa values for Asp, Glu, His and Lys residues using a method that has been calibrated on experimental pKa measurements and achieves a high overall accuracy, but does usually not capture very large shifts of special active site residues (resort to a different method for these and use the pKaRes command to override YASARA's predictions, also look at the pKa prediction result file for scientific details). These calculations are only possible with periodic boundaries and consider all atoms in the simulation cell except waters. Based on the chosen pH, YASARA then assigns the protonation states. Asp and Glu are protonated if the predicted pKa is higher than the pH. His is protonated if the predicted pKa is higher than the pH and it does not accept a hydrogen bond, and deprotonated otherwise. Cys is deprotonated if the selected pH is higher than 8.7. Lys is deprotonated if the predicted pKa is lower than the pH. Tyr and Arg are not modified, because the (Y)AMBER force fields do not have default parameters for the deprotonated states. The pKa predictions are saved in a text file with *.pka extension and can additionally be obtained with the pKaRes command. Then the simulation box is filled with water molecules, and Na/Cl counter ions are placed at the locations of the lowest/highest electrostatic potential until the cell is neutralized and the requested NaCl concentration is reached (normally 0.9%). The location of the counter ions does not matter in practice, since they randomly diffuse away later. If you want to keep YASARA from touching some structurally important waters, fix at least the water oxygen . Finally a short MD simulation is run for the solvent, and water molecules are subsequently deleted until the water density has reached the requested value. Note that the pH setting affects only the initial protonation states but has no additional influence during a simulation. When running simulations of the same protein at different pH values, use the Compare command to make sure that there are indeed different protonation states, otherwise you would simply run the same simulation multiple times.

Example 3:
Experiment Morphing

Prepare a morphing experiment, see example macro below. YASARA transforms one structure into another during a simulated annealing minimization. This is done by applying forces that pull the atoms in the initial (start) structure to their respective coordinates in the target (end) structure. The RMSD to the target is monitored, and the pulling forces are increased when the morphing process gets stuck in a local minimum. Snapshots can be saved to create a movie of the transformation. Morphing essentially requires that initial and target structures contain the same atoms in the same order (i.e. same atom names, residue names and molecule names), nevertheless the target structure may contain fewer atoms. In this case no pulling force will be applied to those atoms that exist only in the initial structure. If you get an error message about atoms missing in the initial structure, use the CompareAtom command to get a complete list of all atoms that differ. While morphing between snapshots from MD simulations is trivial (since these contain exactly the same atoms), morphing between two different PDB files usually requires manual intervention (like renumbering residues or deleting atoms in the target structure that are missing in the initial structure). Most morphing difficulties arise when the initial and target structures are very different (so that atoms must effectively pass through each other to reach the target), or when the target has incorrect geometry (so that e.g. bonds must be stretched on the way to the target). The 'MorphForce' parameter (see example below) provides the means to fine-tune YASARA's behavior in these cases. Note that pulling too strong can lead to frequent simulation failures. To describe this method in a materials & methods section, the following text could be used: "A low-energy pathway between the two conformations was calculated with YASARA[ref] using the .... force field[ref ] with a ... A force cutoff. After removing clashes with an initial unrestrained steepest descent minimization, the procedure continued with a restrained simulated annealing minimization (time step 2 fs, atom velocities scaled down by 0.9 every 10th step). A force of 100 Piconewton (pointing towards the target coordinates) was applied to all the atoms and the RMSD from the target coordinates was measured every 10th step. If the RMSD improved less than 0.01 A, the pulling force was increased by 50%, till a maximum of 10000 pN.".

Example 6:
Experiment NMRFolding

Prepare an NMR folding experiment in YASARA Structure with NMR module, see example macro below. Note that YASARA Structure and the NMR Structure Determination Module are required. YASARA folds a usually linear peptide chain to a globular conformation using distance- and dihedral angle restraints derived from an NMR spectrum. This is done quickly using internal coordinates, i.e. by changing torsion angles. The torsion angles are picked randomly from knowledge based potentials defined by one of the YASARA force fields. During the folding process, the restraints are added sequentially, from short- to long-ranged ones. The result is not accurate, but captures the overall protein topology and can quickly be converted to a realistic structure by molecular dynamics refinement.

Example macro 1:
# EXAMPLE Experiment Morphing
# Requires YASARA Dynamics
Clear
# Load the starting structure
start = LoadPDB 1crn
# Load the target structure and clean both
# (Provide your own target structure here)
end = LoadPDB 1crn_md
Clean
# Create cell and choose force field
Cell Auto
ForceField AMBER03,SetPar=Yes
# Prepare morphing experiment
Experiment Morphing
  # Initial start structure
  StartObj (start)
  # Final end structure
  EndObj (end)
  # Number of intermediate structures to save
  Structures 10
  # Filename for intermediate structures (will be incremented)
  StructureFile morph0000
  # Morphing force in PicoNewton
  #   Min = minimum force (can be increased to speed up morphing)
  #   Switch = force beyond which non-bonded interactions will be gradually switched off
  #   Max = maximum force (morphing stops if it gets stuck and the maximum force has been reached)
  MorphForce Min=100,Switch=2000,Max=10000
# Start experiment
Experiment On
# Wait till end of experiment
Wait ExpEnd

Example macro 2: 
# EXAMPLE Experiment Neutralization
# Requires YASARA Dynamics
# IMPORTANT: Look at the *.pka result file for further details and hints.
Clear
LoadPDB 1crn
# Mutate one arginine to glutamate, net charge is now -2
SwapRes Arg 17,Glu
Clean
Cell Auto
ForceField AMBER96
Cutoff 7.86
Boundary periodic
# Prepare neutralization experiment
Experiment Neutralization
  # Fill the cell with water at a density of 1.0 g/cm^3
  WaterDensity 1.0
  # And NaCl counter ions with 0.9%
  NaCl 0.9
  # Protonate ionizable groups according to pH 7.0
  pH 7.0
  # Save pKa predictions as dat/1crn_mutant.pka
  pKaFile 1crn_mutant
  # Finish quickly (final water density will be OK, but not exact)
  Speed Fast
  # If needed, a residue selection of vacuum in cell coordinates, where no water will be placed.
  # The following example keeps a 30 A membrane layer consisting of POP residues empty:
  # Vacuum LocalY>15 LocalY<45 and with distance<5 from POP
# Start experiment
Experiment On
# Wait till end of experiment
Wait ExpEnd

Example macro 3: 
# EXAMPLE Experiment Minimization
# Requires YASARA Dynamics
Clear
LoadPDB 1crn
Clean
ForceField NOVA
Cutoff 10.48
Boundary Wall
# Prepare energy minimization experiment
Experiment Minimization
  # Stop minimization as soon as the improvement per atom after 200 simulated annealing
  # steps drops below 0.01 kJ/mol or kcal/mol (depending on current EnergyUnit).
  # To skip simulated annealing completely, use 'Convergence immediate'.
  Convergence 0.01
# Start experiment
Experiment On
# Wait till end of experiment
Wait ExpEnd

Example macro 4: 
# EXAMPLE Experiment Docking
# Requires YASARA Structure
Clear
# Load and clean the complex
LoadPDB 1hvr,Download=yes
Stick
Style Ribbon
NiceOri
DelRes HYD
Clean
# Split receptor and ligand
Split
# Join the protein chains in a single object
JoinObj Protein,1
RenumberObj all
# Place the cell around the binding pocket (i.e. the ligand)
RemoveObj 1
Cell Auto,Extension=8
AddObj 1
# Set the force field used to calculate ligand charges
ForceField AMBER99
# Perform the docking
Experiment Docking
  # Use AutoDockLS, a fast 'L'ocal 'S'earch to tune an existing complex,
  # normally you would use AutoDockLGA, the 'L'amarckian 'G'enetic 'A'lgorithm.
  Method AutoDockLS
  # Select receptor object
  ReceptorObj 1
  # Select ligand object
  LigandObj 2
  # Number of docking runs (normally 50-100)
  Runs 1
  # Maximum RMSD allowed for ligands in the same cluster
  ClusterRMSD 5.0
  # Name to save the various complexes, one per cluster, the number
  # at the end of the filename will be incremented
  ResultFile 1hvr_complex001
  # Additional docking parameters specified with 'DockPar' are simply added to
  # the AutoDock input file:
  # Uncomment below to set the number of energy evaluations:
  # DockPar ga_num_evals 25000000
  # Uncomment below to set the number of generations:
  # DockPar ga_num_generations 27000
  # Uncomment below to set the population size
  # DockPar ga_pop_size 150
  # Uncomment below to keep temporary files in the current working directory:
  # TmpFileID YourChoice
Experiment On
Wait ExpEnd
# Show close-up
Pos X=0,Y=4,Z=46
Ori Alpha=-23,Beta=21,Gamma=-6

Example macro 5: 
# EXAMPLE Experiment HomologyModeling
# Requires YASARA Structure
Clear
# Build homology model
Experiment HomologyModeling
  # Specify the target sequence
  SequenceFile target.fasta
  # Maximum oligomerization state
  OligoState 2
  # Number of PSI-BLAST iterations
  PsiBLASTs 6
  # The maximum PSI-BLAST Evalue (minimum can be set to avoid easy models)
  EValue Max=0.5,Min=0
  # Maximum number of templates to consider, and how many with the same sequence
  Templates 3,SameSeq=1
  # Custom templates that are not in the PDB must be loaded before and selected here:
  #TemplateObj all
  # Maximum number of ambiguous alignments to consider per template
  Alignments 1
  # Alignments could also be provided:
  #AlignFile alignments.fasta
  # Number of samples to try per loop (see OptimizeLoop command)
  LoopSamples 50
  # Maximum number of unaligned loop residues to add to N- and C-termini
  TermExtension 10
  # Maximum and minimum number of residues (to avoid very large and small models)
  #Residues Max=1500,Min=30
  # Speed of animations
  Animation Normal
  # The common start for result filenames
  ResultFile model
Experiment On

Example macro 6: 
# EXAMPLE Experiment NMRFolding
# Requires YASARA Structure and the NMR Module
Clear
# Build a linear peptide chain from the file sequence.fasta
BuildMol sequence.fasta
Clean
# We need a forcefield with knowledge based potentials for sampling
ForceField YASARA
SimSteps 500
# Fold the structures from the stretched out conformation
Experiment NMRFolding
  # The object to fold
  StartObj 1
  # The file(s) with NMR restraints (default extension is .tbl)
  RestrainFile restraints.tbl
  # The number of structures to generate
  Structures 1
  # The filename for the structures (digits will be incremented)
  StructureFile ensemble001
# Generate the structure
Experiment On
Wait 59
Style Tube
